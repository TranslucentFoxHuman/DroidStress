# DroidStress
## これは何?
このプログラムはAndroid用のシンプルなストレステストを行なうプログラムです。

このプログラムには以下の機能があります:
- CPUに無駄な計算をさせて、負荷をかけます
- 任意のスレッド数で処理を行ないます
- バックグラウンドで処理を行ない、アプリが閉じられている間も処理を行なえます
- OLEDスクリーンを搭載した端末で画面の焼き付きを防ぐために、真っ黒なブランクスクリーンを表示します
- メモリのストレステストも行えます。

これは私がストレステストプログラムをF-Droidで検索した時、ひとつもストレステストプログラムが存在しなかったことから作成されました。

## 使用方法
Android 13以降では、最初に起動した時に通知の権限を取得するダイアログが表示されます。これはアプリの画面が開かれていない後も処理を継続するために必要です。

「Threads:」入力欄には、処理を行なうスレッド数を入力します。

「Duration (minute):」入力欄には、処理を行なう時間を分単位で入力します。この入力欄に数字が入っている場合、その数字の時間の分だけ処理を行ないます。無限に処理を続ける場合は内容を空欄にしてください。

「Memory stress test (MiB:)」入力欄は、メモリのストレステストも行なう場合にメビバイト単位で数値を入力します。入力されたサイズのメモリを確保、適当な値の書き込み、領域の開放を繰り返すことでメモリに負荷をかけます。

「Start」ボタンを押すと、処理が開始されます。処理が開始されると、バックグラウンドで処理中である事を示す通知が表示されます。
「Stop」を押すと処理を停止します。

「Show blank screen」を押すと、真っ黒な画面を表示した上で、画面がスリープするのを一時的に停止します。OLEDディスプレイを搭載したデバイスで、長時間ストレステストを行なう場合に便利です。

メニューバーの「About...」を押すと、アプリのバージョンとライセンス情報を表示します。

### メモリのストレステスト
v1.6.0以降、メモリのストレステストも行う機能が追加されました。
「Memory stress test (MiB):」入力欄の数値のサイズの分メモリを確保し、それに適当な値を書き込み、その確保した領域の開放を繰り返すことでメモリに負荷をかけます。  
メモリのI/Oの待ち時間が発生するため、CPUにはあまり負荷がかからなくなるかもしれません。

「Memory stress test (MiB:)」入力欄の数値は理想的には1〜64が望ましいでしょう。それより大きな値では上手くメモリを確保出来なかったり、メモリ確保の待ち時間が増大しCPUに負荷がかからなくなったりします。

Androidでは各アプリが使用できるメモリの量に制限を課しています。多くの場合でそれは512MiBです。仮にメモリにより多くの空きがあったとしても、この制限を越えてメモリを確保することはできません。そのためこの制限のもとで利用可能なメモリサイズより大きい値が入力された場合、利用可能なストレステストで確保するメモリサイズはその制限の値に設定されます。

メモリサイズに大きな値を指定すると「Memory is full!」というトースト通知が表示されるかもしれません。これはメモリストレステストが指定された分のメモリを確保出来なかった場合に表示されます。Android OSの制限よりも小さいメモリを指定しても表示されることがあるでしょう。  
これはメモリ管理の仕組みに起因します。任意のタイミングでプログラマがメモリを開放出来るC言語などと異なり、Kotlin/Javaでは不要になったメモリがすぐには開放されず、OS(VM)がリソースの使用状況に基いて自動的に開放します。そのため前回のサイクルで確保したメモリ領域がまだ残っており、その状態でさらにメモリを確保しようとするため、メモリ制限の値より低い値でもメモリが確保出来なくなります。

## ユースケース
- 電池のリフレッシュ  
  電池の残量が正しく表示されない場合、一度100%まで充電してから0%まで放電すると、残量の測定が補正されることがあります。このアプリはそのような作業でバッテリーを消費させるのに役立ちます。
- 携帯電話を温める  
  携帯電話本体の温度が低下すると、突然シャットダウンしたり、フリーズしたりと、動作が不安定になります。冬場、デバイスが不安定になる前にこのプログラムを使用して、デバイスを温め動作を安定させましょう。
- デバイスで暖をとる  
  Snapdragon 810のようなCPUを搭載したデバイスをお持ちですか? デバイスを使って暖をとるのにぴったりです! このアプリをデバイスで使用して、ポケットに入れましょう。手を温めることができます。

## あなたの助けが必要です!
このプロジェクトは一人で開発しました。私には実現が難しい以下の要素が不足しています:
- より優れたアプリアイコン  
  アイコンはGPLと互換性のあるライセンスで、Inkscapeで編集可能なSVG形式である必要があります。AIに作成させないでください! AIに作成させる位であれば誰にだって出来ます。
- よりCPUに負荷をかける事のできる計算アルゴリズム  
  現状、とても単純な計算でCPUに負荷をかけています。よりCPUに強い負荷をかけることの出来る計算アルゴリズムが必要です。アルゴリズムはKotlinを始めJava、C言語など、様々な言語で同じアルゴリズムが実行でき、また外部のライブラリに依存せず、特定のアーキテクチャに依存しないものである必要があります。
